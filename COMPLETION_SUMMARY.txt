### 4. Configuration System

All problems have YAML configs with:
- Network architecture (depth, width, activation)
- Training parameters (epochs, learning rate, scheduler)
- Physics parameters (nu, alpha, c, etc.)
- Visualization options

```yaml
# Example: config/wave_1d.yaml
problem: "wave_1d"
network:
  input_dim: 2
  hidden_layers: [64, 64, 64, 64, 64]
  output_dim: 1
  activation: "tanh"
training:
  epochs: 10000
  learning_rate: 0.001
  lr_scheduler: "exponential"
  batch_size: 1024
physics:
  c: 1.0  # Wave speed
```

### Programmatic Usage
```python
from core.networks import PINNNet
from core.pinn_solver import PINNSolver
from problems.forward.wave_1D import Wave1DForward

device = "cuda" if torch.cuda.is_available() else "cpu"
problem = Wave1DForward(c=1.0, device=device)

model = PINNNet(input_dim=2, hidden_layers=[128, 128, 128])
solver = PINNSolver(model, problem, device=device)

data = problem.generate_data(n_samples=1024)
history = solver.train(data, epochs=10000)

from utils.plotting import plot_loss, plot_result_1d
plot_loss(history)
plot_result_1d(model, device=device)
```
